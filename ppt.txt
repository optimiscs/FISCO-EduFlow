
​   各位评委老师大家好，我们团队设计的项目是“学链通—基于FISCO区块链的学历全流程管理系统”。 ->


​   学链通是一个基于fisco区块链技术的学历全流程管理系统，立足于区块链技术赋能教育新基建的国家战略方向，应用于跨境学历互认、政企人才快速核验、个人教育数据主权管理等高价值场景，致力于保障人才流动中的可信认证与隐私安全。->


​   有数据显示，我国近三年来高等教育在学规模逐年增长，连年增长的在学规模意味着全国高等教育学历信息数据库规模越来越大，急需一个轻量化、非中心化的学历管理系统。
​   但与此同时 ->
​   社会上的学历造假、隐私泄露事件层出不穷，国外院校学历学位认证漏洞频出，急需一个能够实现可信认证、隐私保护的全流程学历管理系统；
​   面对这些社会问题 ->
​   国家已发布多项政策，不断更迭和完善学历认证系统，提升数字化公共服务水平，完善全国统一、分级使用、开放共享的教育信息管理系统 。->
​   当下采用的学历可信管理方案具有过度依赖第三方认证机构、文件兼容性差、隐私保护能力有限、中心化系统鲁棒性差的缺陷，可见目前的学历可信管理方案仍然有待完善。->
​   综上，目前学历认证领域具有易造假、难防伪、弱隐私的三大痛点，无法满足学历可信全过程认证，也无法全方位保障人才流动过程中的可信认证与隐私安全 ->


​   2021年， 区块链 正式成为十四五规划中的七大发展产业之一。这标志着 区块链 升级为国家战略。2023年，我国区块链专利申请量、授权量持续位居全球首位。区块链产业园区遍布发达省市，FISCO BCOS、ChainMaker等区块链底层项目陆续开源。这为新型学历认证系统的开发提供了技术支持。->
​   结合区块链技术“去中心化”的数据库的本质与去中心化、容错性强、匿名性强、不可篡改和可追溯的特点，我们发现区块链技术能够完美契合建设数字化社会、建立全过程可信、隐私保护完善的学历管理认证系统的需求。 ->

​   因此，我们结合区块链、CA体系、零知识证明技术设计了对应的解决方案——一套能够保障数据真实性和完整性、增强隐私保护，防止信息泄露、具有高系统可信度和防伪能力的学链通全流程可信学历认证管理系统 。 ->


​   接下来我将为各位评委老师介绍一下我们的核心技术路线。
​   我们采用“主链+侧链”的双链混合模式，结合“链下状态通道”，设计三类区块链通道，保障系统能够在不需要每个参与节点均验证网络中的每项操作的情况下工作。主链采用了联盟链进行构建，不同主体的节点设置不同的权限，兼顾公有链和私有链的特点，实现数据分级管理和权限分层授予。利用区块链的特性解决学历全流程管理造假问题的同时，还必须考虑系统开销问题。为了解决这个问题，我们设计了侧链，用于对区块链进行扩容，以满足各节点上链需求，降低系统开销。 ->
​   冗余的架构会大大降低区块链处理交易的速度，也使得系统部署需要的资源庞大，费用昂贵。为了解决这个问题，我们引入了链下状态通道，负责小额多次的认证收费、数字人民币交易的合约部署，实现链下交互、链上清算，加快交易速度，提高交易隐私性。 ->
​   本作品基于MerkleTree算法设计了主侧链数据同步、数据验证的智能合约。在学生录入信息以后生成的hash作为用户的“区块链身份证”。从叶子结点向上开始两两计算hash值，直至得到根哈希MerkleRoot，通过MerkleRoot的值不变来保障学历认证系统的不可篡改性。 ->
​   根据学位、学历认证智能合约模型，结合传统学位学历认证流程，本作品提出了学位学历认证智能合约算法。当学生与用人单位就学历认证需求达成共识时，先由用人单位向用户提供认证需求，然后学生确认输出的个人数据，智能条约将需要转发的个人信息经过模糊化处理和哈希处理生成包含“区块链身份证” 的PDF文件，最后用人单位接受PDF文件，并可对其验证溯源。->


​   本作品为了更好地对系统进行测试，基于 MSVC 和开源 GUI 库DearImGui 自研了可视化数据采集引擎Hazel Engine， 能够实现对于某一个封装包或进程接口的监听，将目标数据可视化至ImGui的Inspector，也能够输出profile 文件，具有方便、简易，学习成本低和效率高、系统简洁，结果稳定真实的特点。 ->


​   总的来说，我们采用国产化fisco，设计了区块链架构，系统共包含智能合约、共识机制、零知识证明、CA认证四大模块，通过web管理系统、“区块链浏览器”，便于学生、管理者、第三方单位等节点进行操作管理，形成一个完整链条。
​   学生数据通过数据处理算法和零知识证明协议，形成一张“强隐私保护”的“区块链身份证” 写入“一链通”，形成链上数据。由学生、教育部门、协查机构和第三方单位四类节点，构成主链、 侧链、和状态通道三类链条，通过共识算法和智能合约保障区块链良好运行，通过CA认证和数据分级处理，进行节点管控和学历全流程管理电子防伪，通过“二维码”模式进行纸质防伪，能够实现一键转发与批量验证操作、数字货币交易、学历防伪等功能，满足多功能需求与不同的使用场景，做到“隐私保护”和“防伪”两大痛点的保障。->
​   在智能合约模块，我们以fisco chain为底层技术设计了学籍信息上链，数据同步验证以及学历全流程管理认证三个核心合约。
​   在学籍信息上链合约中，学生在首次登录到系统后，需要填写自己真实的个人信息。这些信息将交由根据教育部节点承担主要审核工作,审核成功后经过数据会进行加密，通过合约上链并储存在联盟主链。
​   在数据同步、验证合约中，我们基于MerkleTree构造了了主侧链数据同步、数据验证的智能合约。在学生录入信息以后，使用MerkleTree 算法进行哈希锁定，生成一串hash值作为用户的“区块链身份证”，并将得到的MerkleRoot同步到以联盟侧链。一旦学籍信息被篡改，Merkle Root 也会随之改变。
​   在学历全流程管理认证合约中，在得到用户与用人单位的双边认证之后，将需要转发的个人信息经过模糊化处理和哈希处理生成包含“区块链身份证” 的PDF文件，用人单位接受PDF文件，并可对其验证溯源。 ->
​   在共识机制模块中，我们通过各类共识算法来实现分布式节点间数据的一致性和真实性。在整个区块链系统中，两种类型的节点：普通节点和超级节点。我们通过通过可信硬件（TEE）认证普通节点的参与者身份。而超级节点在服务器上运行，数量少，主要进行存储等资源消耗量大的工作，普通节点根据需要从超级节点获取储账本和密钥值等数据。 ->
​   我们考虑到节点安全性和电子认证的缺陷，决定引入CA认证保障安全性，提高防伪性能。通过证书颁发机构发放和管理数字证书，并作为电子商务交易中受信任的第三方，承担公钥体系中公钥的合法性检验的责任。 ->
​   “过度分享信息”往往会导致隐私泄露问题，带来了极大且隐秘的损害。我们通过通过零知识证明和智能合约配合，对涉及学生个人隐私的敏感信息数据进行分解，建立特殊用途使用的数据结构，以利确保外界无法将其中的某些敏感信息片段抽取出来单独使用。 ->
